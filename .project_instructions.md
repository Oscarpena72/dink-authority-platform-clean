## Universal Video Module
- **Component**: `components/universal-video-module.tsx` — single reusable component that auto-detects platform from URL
- **Supported platforms**: YouTube, Instagram, TikTok, Facebook
- **YouTube rendering**: 16:9 horizontal, click-to-play thumbnail → inline iframe embed
- **Instagram/TikTok/Facebook rendering**: 9:16 vertical editorial card with poster image, gradient border, play button overlay, platform branding, "Ver en [Platform]" CTA that opens in new tab (Instagram blocks iframe embeds)
- **Exported utilities**: `detectPlatform(url)` returns `'youtube'|'instagram'|'tiktok'|'facebook'|null`, `VideoPlatform` type
- **Database field**: `videoUrl String?` added to Tip, Junior, and Article models (unified field replacing old `youtubeUrl` and `instagramVideoUrl`)
- **Backward compatibility**: Old fields (`youtubeUrl` on Tip, `instagramVideoUrl` on Junior) kept in schema. Frontend falls back: `videoUrl ?? youtubeUrl` for Tips, `videoUrl ?? instagramVideoUrl` for Juniors
- **CMS admin forms**: All three (Tips, Juniors, Articles) have universal "Video URL" input accepting YouTube, Instagram, TikTok, or Facebook URLs
- **API routes**: All six endpoints (create+update for Tips, Juniors, Articles) handle `videoUrl`
- **i18n keys**: `video.sectionTitle`, `video.play`, `video.watchOn`
- **Data migrated**: Existing `youtubeUrl` values copied to `videoUrl` for Tips; existing `instagramVideoUrl` values copied to `videoUrl` for Juniors
- **Video Poster Image**: `videoPosterImage String?` field on all three models. CMS shows "Video Poster Image" input (optional). If empty, frontend falls back to featured/main image. Logic: `videoPosterImage || featuredImage`
- **Pro Players**: NOT a separate model/route — just articles with `category: 'pro-players'`. Access via `/articles?category=pro-players`. Video module works automatically since Articles already have UniversalVideoModule.
