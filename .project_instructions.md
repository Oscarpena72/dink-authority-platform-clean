## PWA (Progressive Web App)
- **manifest.json**: Already existed in `public/` with icons 192x192 and 512x512, standalone display, brand-purple theme
- **Service Worker** (`public/sw.js`): Network-first for pages (with offline fallback), cache-first for static assets, skips API routes
- **Offline page** (`public/offline.html`): Branded offline fallback with retry button
- **SW Registration** (`_components/pwa-register.tsx`): Registers SW on mount
- **Install Prompt** (`_components/pwa-install-prompt.tsx`): Smart banner for Android/desktop (uses beforeinstallprompt) + iOS guide modal (step-by-step Safari instructions). Dismissible for 24h, hidden if already standalone.
- Both PWA components loaded in root `layout.tsx`

## Phase 3A — Subscriber Capture (Email + Phone)
- **New model**: `Subscriber` (id, email unique, phoneNumber optional, source, subscribedAt) — separate from existing `NewsletterSubscriber`
- **API**: `POST /api/subscribers` (public, upsert by email) + `GET /api/subscribers` (admin, supports `?format=csv` export)
- **Inline subscribe block**: Inserted at ~37% of article content (both desktop and mobile) via `SubscribeForm` component with source="article"
- **Footer subscribe form**: Added as 4th column in footer grid via `SubscribeForm` component with source="footer"
- **Admin page**: `/admin/subscribers` with table, search, refresh, CSV export button
- **Admin nav**: "Subscribers" item added with Users icon between Newsletter and Settings

## Phase 3B — SEO / Discover Engine
- **robots.ts**: Dynamic, allows /articles, /magazine, /about, /contact; disallows /admin, /api, /login; references sitemap
- **sitemap.ts**: Dynamic, includes all published articles (priority 0.8), magazine editions (priority 0.7), static pages (/, /articles, /magazine, /about, /contact)
- **Article page metadata** (`articles/[slug]/page.tsx`):
  - Canonical URL via `alternates.canonical`
  - `robots: { index: true, follow: true, 'max-image-preview': 'large', 'max-snippet': -1, 'max-video-preview': -1 }`
  - Complete OpenGraph: type article, publishedTime, modifiedTime, authors, section, locale
  - Twitter Card: summary_large_image with site handle
- **JSON-LD Structured Data**: Two schemas injected in article pages:
  - `NewsArticle` with headline, image, datePublished, dateModified, author, publisher (with logo), description, mainEntityOfPage
  - `BreadcrumbList` with Home → Articles → Article Title
- **Visible breadcrumbs**: Replaced "Back to Articles" link with `Home / Articles / Title` breadcrumb trail in article-detail-client.tsx
- **Global max-image-preview**: Added `<meta name="robots" content="max-image-preview:large" />` in root layout.tsx head
