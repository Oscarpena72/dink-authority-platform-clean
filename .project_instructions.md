## Deployment Pipeline
- **Production domain**: `dinkauthoritymagazine.com` (served via Vercel)
- **Vercel project**: `dink-authority-platform-clean`
- **GitHub repo**: `Oscarpena72/dink-authority-platform-clean`
- **CRITICAL: TWO BRANCHES** ‚Äî `main` (Vercel deploys from THIS) and `master` (local default). Always push to BOTH: `git push origin master` then `git push origin master:main`
- **Deployment flow**: Code changes ‚Üí `git push origin master && git push origin master:main` ‚Üí Vercel auto-deploys from `main` to `dinkauthoritymagazine.com`
- **CRITICAL**: Always push to GitHub after changes. Abacus deployment alone does NOT update the production site.
- **Vercel build command**: `node scripts/fix-prisma-for-vercel.js && npx prisma generate && next build`
- **Vercel install command**: `npm install --legacy-peer-deps`

## Leads, CRM & Contact Registration System (April 2026)

### Data Models with Source Tracking
- **NewsletterSubscriber**: email, name?, source (default: "homepage"), createdAt
- **Subscriber**: email, phone?, name?, source (default: "article"), createdAt  
- **ContactSubmission**: name, email, message, phone?, source (default: "contact-form"), status (default: "new"), createdAt

### Admin Pages
- `/admin/newsletter` ‚Äî Search, source column with colored badges, CSV export
- `/admin/subscribers` ‚Äî Search, name+source columns, CSV export
- `/admin/contacts` ‚Äî **NEW** expandable card view, status management (new/read/replied/archived), search, CSV export
- `/admin/leads` ‚Äî **NEW** unified CRM overview: stats cards, source breakdown, recent activity timeline, links to sub-modules, Export All CSV

### APIs
- `GET /api/newsletter?format=csv` ‚Äî Auth-protected, CSV export
- `GET /api/contact?format=csv` ‚Äî Auth-protected, CSV export  
- `GET /api/subscribers?format=csv` ‚Äî Auth-protected, CSV export
- `PUT/DELETE /api/contact/[id]` ‚Äî Status update and delete

### RBAC
- `/admin/contacts` and `/admin/leads` require `super_admin` or `admin` role

### Navigation
- Sidebar order: Newsletter ‚Üí Subscribers ‚Üí Contacts ‚Üí Leads & CRM ‚Üí Users ‚Üí Settings

## Brevo (Sendinblue) Integration (May 2026)

### Configuration
- **Brevo API Key**: stored in `.env` as `BREVO_API_KEY`
- **List ID**: 3 (env: `BREVO_LIST_ID`)
- **Sender**: `contact@dinkauthoritymagazine.com` (env: `DEFAULT_FROM_EMAIL`, verified in Brevo)
- **CRITICAL**: `.env` is in `.gitignore` ‚Äî Brevo env vars must be added separately in Vercel dashboard

### Architecture
- `lib/brevo.ts` ‚Äî Server-side utility: syncContactToBrevo, syncBatchToBrevo, sendWelcomeEmail, sendArticleNotification, getBrevoListStats
- Direct Brevo REST API calls (no SDK), auth via `api-key` header

### Subscriber Sync
- New subscribers auto-synced to Brevo list on creation (fire-and-forget)
- Welcome email sent to new subscribers (dark-themed HTML, neon green accents)
- Both `Subscriber` and `NewsletterSubscriber` models sync to same Brevo list
- `/api/brevo/sync` ‚Äî Admin-only bulk sync of all existing subscribers (deduplicates)
- "Sync to Brevo" button on `/admin/subscribers` page

### Article Notifications
- `/api/brevo/notify` ‚Äî Creates Brevo email campaign and sends immediately
- Anti-duplicate: checks `notificationSentAt` field before sending
- Article schema fields: `notificationSentAt`, `notificationChannel`, `brevoCampaignId`
- "Notify Subscribers" UI in article editor sidebar (published articles only)
- Channel selection: Email (active), SMS/Both (coming soon, disabled)
- Includes message preview and confirmation dialog
