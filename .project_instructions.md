## Brevo (Sendinblue) Integration (May 2026)

### Configuration
- **Brevo API Key**: stored in `.env` as `BREVO_API_KEY`
- **List ID**: 3 (env: `BREVO_LIST_ID`)
- **Sender**: `contact@dinkauthoritymagazine.com` (env: `DEFAULT_FROM_EMAIL`, verified in Brevo)
- **CRITICAL**: `.env` is in `.gitignore` — Brevo env vars must be added separately in Vercel dashboard

### Architecture
- `lib/brevo.ts` — Server-side utility: syncContactToBrevo, syncBatchToBrevo, sendWelcomeEmail, sendArticleNotification, getBrevoListStats
- Direct Brevo REST API calls (no SDK), auth via `api-key` header

### Subscriber Sync
- New subscribers auto-synced to Brevo list on creation (fire-and-forget)
- Welcome email sent to new subscribers (dark-themed HTML, neon green accents)
- Both `Subscriber` and `NewsletterSubscriber` models sync to same Brevo list
- `/api/brevo/sync` — Admin-only bulk sync of all existing subscribers (deduplicates)
- "Sync to Brevo" button on `/admin/subscribers` page

### Article Notifications
- `/api/brevo/notify` — Creates Brevo email campaign and sends immediately
- Anti-duplicate: checks `notificationSentAt` field before sending
- Article schema fields: `notificationSentAt`, `notificationChannel`, `brevoCampaignId`
- "Notify Subscribers" UI in article editor sidebar (published articles only)
- Channel selection: Email (active), SMS/Both (coming soon, disabled)
- Includes message preview and confirmation dialog

### SMS Integration (May 2026) — SIMPLIFIED
- **Architecture**: SMS recipients managed entirely in Brevo (List ID 3), NOT in local DB
- **No admin SMS UI**: `/admin/subscribers` is email-only (original version restored)
- **No `/api/sms/` routes**: Deleted. All SMS goes through `/api/brevo/notify`
- Env vars: `BREVO_SMS_SENDER=DINKAUTH`, `SMS_ENABLED=true`, `BREVO_API_KEY` (needed for SMS)
- `lib/brevo.ts` functions: `isSmsEnabled()`, `sendTransactionalSms()`, `sendBulkSms()`, `buildArticleSmsMessage()`, `getBrevoListPhoneNumbers()`
- **Phone source**: `getBrevoListPhoneNumbers()` fetches contacts from Brevo List ID 3 via `GET /v3/contacts/lists/{listId}/contacts`, extracts `attributes.SMS`, filters `smsBlacklisted`, normalizes to international format
- **Flow**: Article editor → Notify Subscribers → SMS/Both → API fetches phones from Brevo list → sends transactional SMS to each
- Article form: SMS and Both radio buttons enabled
- `SmsContact` model still in schema.prisma (harmless, unused by current code)
- **CRITICAL**: `BREVO_API_KEY` must be in preview `.env` for testing; already in Vercel dashboard for production
